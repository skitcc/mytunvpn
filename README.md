
**MyTunVPN** — легковесный VPN-туннель на уровне L3 (сетевой уровень), написанный на Go. Проект создан для изучения работы TUN-устройств, инкапсуляции IP-пакетов в UDP и низкоуровневой маршрутизации в Linux.



## Требования

- ОС: **Linux** 
- Права: **Root** 
- Установленный **Go 1.20+**.

## Запуск

### 1. Сборка проекта

```bash
go build -o vpn_server ./server/main.go

go build -o vpn_client ./client/main.go
```

### 2. Запуск Сервера (на VPS)

Запустите сервер. Он создаст интерфейс и назначит ему IP 10.1.0.20.

```bash
sudo ./vpn_server
```

### 3. Запуск Клиента (на локальном ПК)

Укажите публичный IP адрес вашего сервера в конфигурации или коде.

```bash
sudo ./vpn_client
```

## Диагностика и маршрутизация

Чтобы пакеты корректно проходили через туннель, используйте следующие команды:

### Проверка интерфейса

Убедитесь, что интерфейс поднялся и получил IP (например, tun1):

```bash
ip addr show dev tun1
```

### Проверка маршрутов

Ядро должно знать, что трафик для 10.1.0.20 идет в туннель:

```bash
ip route show | grep tun
```

### Тестовый пинг

Для стабильной работы на ранних этапах используйте флаг -I, чтобы ОС выбрала правильный Source IP (10.1.0.10):

```bash
ping -I tun1 10.1.0.20
```

## Исправление потерь пакетов (Reverse Path Filter)

Если логи показывают активность, но ping не видит ответа, отключите фильтр обратного пути в Linux:

```bash
sudo sysctl -w net.ipv4.conf.all.rp_filter=0
sudo sysctl -w net.ipv4.conf.default.rp_filter=0
sudo sysctl -w net.ipv4.conf.tun1.rp_filter=0
```
